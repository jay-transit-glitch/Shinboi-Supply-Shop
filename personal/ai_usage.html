Documenting Efficiency: The Role of AI in Project Development

With such an complex website, I used to help of Gemini. I treated the AI not just as a tool, but as a cruical resource management strategy. It helped with solving problems in my code that I just couldn't understand why it didn't work.

By using AI to handle the tedious scaffolding and complex transaction logic, I freed up my limited time to focus on the things that actually matter for my final grade, such as layout design, accessibility, and unique project structure.

Specific Contributions by Google Gemini

1. Crtitcal JavaScript Debugging
AI was used to diagnostic and resolve several major functional errors in the project's JavaScript files:
  • **Cart Display:** Corrected a critical typo in `cart.js` that used the non-existent variable `item.Total` instead of the local variable `itemTotal` when calculating and displaying cart rows.
  • **Form Validation:** Fixed a `ReferenceError` in `form_validation.js` where the function used the undefined variable `feedbackSpan` instead of the correctly initialized variable `feedbackInput`, which was preventing form submission.
  • **HTML Syntax:** Identified missing opening bracket on the quantity `` tag in `cart.js`, which caused the input box to render as plain text.
2. Asset Generation
All major visual assets for the site, including product photos and promotional images, were generated using Google Gemini's image generation capabilities.
  • Product Image Example: The Basic Kunai Set image was created via prompt to ensure unique, high-quality, non-copyrighted imagery
  • Styling/Theming: Assisted in generating cohevise color palettes and Specific CSS properties for shadows and transitions.
3. Documentation and Planning
AI assisted in project planning and presentation structure.
  • HTML Structure: Assisted with boilerplate structure and accessibility checks (e.g., proper use of `aria-label` and semantic HTML tags).
